# Pool Cleanup Notes

This file is generated by `npm run db:pool-cleanup-sql` and contains warnings and manual steps required for database cleanup.

## AUTHORITATIVE CLEANUP (One-Time)

For the authoritative, one-time cleanup, use:
```bash
npm run db:authoritative-cleanup-sql
```

This generates:
- `analytics/reports/pool_cleanup_corpus_authoritative.sql` (run ONLY on CORPUS)
- `analytics/reports/pool_cleanup_runtime_authoritative.sql` (run ONLY on RUNTIME)
- `analytics/reports/pool_cleanup_verification.sql` (run against both DBs separately)

## Run Order (Authoritative Cleanup)

1. **Generate authoritative cleanup SQL**: `npm run db:authoritative-cleanup-sql`
   - Generates deterministic cleanup SQL matching PSA doctrine

2. **Review SQL files**
   - Verify operations match expected cleanup
   - Check that CORPUS SQL only affects CORPUS tables
   - Check that RUNTIME SQL only affects RUNTIME tables

3. **Apply SQL to databases**
   - Run `pool_cleanup_corpus_authoritative.sql` against CORPUS database **ONLY**
   - Run `pool_cleanup_runtime_authoritative.sql` against RUNTIME database **ONLY**
   - **CRITICAL**: Never run CORPUS SQL on RUNTIME or vice versa

4. **Run verification queries**
   - Execute `pool_cleanup_verification.sql` against CORPUS
   - Execute `pool_cleanup_verification.sql` against RUNTIME
   - Verify all checks return expected results

5. **Final audit**
   - Run: `npm run db:audit`
   - Confirm: duplicates = 0, mapping errors = 0

## Hard Rules After Cleanup

- **NO TABLE MAY EXIST IN BOTH POOLS**
- Promote logic:
  - READS from CORPUS.ofc_candidate_queue
  - WRITES approval state in CORPUS
  - CREATES OFCs in RUNTIME ONLY WHEN ATTACHED TO:
    - assessment_id + question_canon_id + assessment_response(answer=NO)
- MODULE inventory stays in CORPUS
- ASSESSMENT OFCs live in RUNTIME and MUST answer a NO

## Standard Cleanup Process (Alternative)

For standard cleanup using diff-based analysis:

1. **Generate diff report**: `npm run db:pool-diff`
2. **Generate cleanup SQL**: `npm run db:pool-cleanup-sql`
3. **Review generated SQL and notes**
4. **Apply SQL to databases**
5. **Verify cleanup**: `npm run db:audit`
