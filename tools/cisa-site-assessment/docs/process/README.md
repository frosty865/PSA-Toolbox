# Process Directory

## Purpose

This directory contains documentation for **psa-rebuild-specific processes**: UI workflows, presentation logic, and component architecture.

## What Belongs Here

### docs/process/ui/
- UI component architecture
- Component behavior and interactions
- Component design patterns

### docs/process/workflows/
- User workflow documentation
- Navigation patterns
- User journey documentation

### docs/process/presentation/
- Data presentation logic
- Formatting and display rules
- Visual design documentation

## What Does NOT Belong Here

- **Doctrine content**: Doctrine belongs in `psa_engine` or `docs/doctrine/` (read-only)
- **Coverage computation**: Coverage belongs in psa_engine
- **Ingestion processes**: Ingestion belongs in psaback

## Authority

**psa-rebuild IS authoritative for process documentation.**

This directory documents how psa-rebuild performs its core functions:
- UI workflows
- Presentation
- Component architecture

## Classification

- **Type**: Process documentation
- **Authority**: psa-rebuild
- **Modifiable**: Yes (for psa-rebuild processes only)
- **Purpose**: Document psa-rebuild-specific operations
