================================================================================
  Infrastructure Dependency Tool (IDT) — Instructions for field personnel
================================================================================

WHAT THIS IS
------------
This folder is a self-contained copy of the IDT web application. You do not need
to install Python, Node.js, or any web server. You only need a supported web
browser (Microsoft Edge, Google Chrome, or Mozilla Firefox — current versions).


WHAT TO KEEP TOGETHER
---------------------
Do not delete, rename, or move folders inside this package. The application
needs every folder that came with it, especially the "_next" folder (it holds
the program code). If "_next" is missing or in the wrong place, the screen may
stay blank.

You may copy the entire folder to your computer or a network drive, but always
copy the whole tree, not single files.


HOW TO OPEN THE TOOL
----------------------
1. Unzip the package (if you received a .zip file) to a folder you can find,
   for example:  Documents\IDT-field  or  Desktop\IDT-field

2. Open the folder and locate the file named:
       index.html
   (It sits at the top level of the folder, next to this instructions file.)

3. Start the tool in one of these ways:
   • Double-click index.html, or
   • Right-click index.html → Open with → choose Edge, Chrome, or Firefox, or
   • In the browser: File → Open File… → select index.html

4. If your organization blocks opening local files, use the method your IT
   department provides for approved offline web apps (some sites publish the
   same folder on an internal website — that is also supported).


FIRST-TIME USE
--------------
• You may see a federal / legal disclaimer. Read it and accept to continue.
• Your work is stored in the browser session on this computer (not on a central
  server) unless your agency configures something different.


DAY-TO-DAY WORKFLOW (typical)
------------------------------
• Use the menus and links inside the application to move between sections
  (for example: categories, dependencies, review).
• Save your work using the options the tool provides (draft export, revision
  package, final export) according to your agency’s process.
• Final export in this field package is usually a canonical JSON file unless
  your IT team has configured a hosted service for Word (DOCX) reports.


IF THE PAGE IS BLANK OR NOTHING LOADS
---------------------------------------
1. Confirm you opened index.html from inside the folder you unzipped — not a
   copy that was moved without the "_next" folder.

2. Turn JavaScript ON in your browser (the tool requires it).

3. Try another supported browser.

4. Ask IT whether a strict security policy blocks scripts on local (file) pages.
   They may need to allow this application or host the folder on HTTPS.

5. Do not rely on opening only one file from email — always use the full folder.


SECURITY AND DATA
-----------------
• Treat exports and assessments according to your agency’s classification and
  handling rules.
• This field build does not require Python or a local web server; it is
  intended for locked-down machines where only a browser is available.


NEED HELP?
----------
Contact your IT point of contact or the program office that distributed this
package. Technical staff can refer to FIELD_DEPLOY_README.txt in this folder
for deployment and hosting notes.
